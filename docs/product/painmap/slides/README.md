# PainMap 破壞式創新教學簡報 — 投影片架構

> **風格**：Steve Jobs Keynote — 極簡、張力、敘事驅動
> **時長**：60 分鐘（含 12 分鐘課堂練習）
> **結構**：Gagné's 9 Events of Instruction
> **目標**：初學者走出教室後 72 小時內發出第一封預售訊息

---

## 賈伯斯風格設計基因

| 基因 | 在簡報中的體現 |
| :--- | :--- |
| **極簡句式** | 短句。一句一行。留白即力量。 |
| **對比張力** | 先立敵人（代理動作），再揭英雄（物理量） |
| **數字具象化** | 不說百分比，說「3 小時」「NT$800」「100 倍」 |
| **反問蓄勢** | 「誰需要？」「跟真人說過話嗎？」停頓。讓答案自己來。 |
| **重複錘擊** | 「去做吧」首尾呼應。「代理動作」反覆出現。 |
| **懸念揭幕** | 每頁先丟問題或衝突，再給答案 |
| **口語敘事** | 講師備註全是口語。像在聊天，不像在上課。 |

## 每頁檔案結構

| 區段 | 說明 |
| :--- | :--- |
| **Action Title** | 結論句標題。觀眾看到的第一句話。 |
| **內容** | 投影片文字。短句。對比。留白。 |
| **視覺設計** | 極簡排版。深色背景。逐幕 build。 |
| **講師備註** | 口語講稿。節奏標記。停頓指示。互動時機。 |
| **過場** | 一句話銜接。不囉唆。 |

---

## 演講節奏指南

| 技巧 | 用法 |
| :--- | :--- |
| **停頓** | 重要句子後停 3-5 秒。讓觀眾自己消化。 |
| **反問** | 問完不等答案。自己接。「跟真人說過話嗎？零。」 |
| **Build** | 一次只出現一個概念。不要一頁全亮。 |
| **數字** | 先出數字（巨大），再解釋。「90%。」停頓。然後說故事。 |
| **重複** | 「代理動作」至少出現 5 次。「物理量」至少 4 次。「去做吧」首尾。 |
| **速度變化** | 講故事慢。講工具快。講結論——最慢。 |
