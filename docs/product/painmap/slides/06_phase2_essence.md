# Slide 06 — Phase 2 問題本質拆解（Gagné Event 4：看清問題下）

---

## Action Title

**Phase 2：十個痛點背後，可能只有一個問題。**

## 內容

你從 Phase 1 拿到了 5 個、10 個痛點。

看起來都不一樣。但如果你往深挖——

很多時候，它們是同一件事。

```
痛點 #1：賣家花太多時間回 LINE
痛點 #3：賣家漏單因為訊息太多        →  同一個問題：
痛點 #5：賣家沒時間做新品                「重複性客服勞動」
```

Phase 2 幫你做聚合。把表面不同的痛點，歸到同一個根。

---

然後，最重要的工具登場。

### 物理量畫布。

四個格子。四個問題。全部要你自己填。AI 填不了。

```
┌────────────────────────┬────────────────────────┐
│ #1 具體的人              │ #2 正在付錢的痛          │
│                         │                         │
│ 我能說出 3 個人名嗎？    │ 他們現在花多少錢/時間     │
│ 能打電話給他們嗎？       │ 在解決這個問題？          │
├────────────────────────┼────────────────────────┤
│ #3 手工可交付             │ #4 收款管道              │
│                         │                         │
│ 不寫程式，               │ 我有現成的方式            │
│ 今晚能幫一個人做一次嗎？  │ 收到錢嗎？               │
└────────────────────────┴────────────────────────┘
```

為什麼叫「物理量」？

因為這四個東西**可以測量**。不是意見。不是感覺。是事實。

你認識幾個人——數得出來。
他們花多少錢——查得到。
你能不能做一次——試了就知道。
你能不能收錢——開個帳戶就行。

**怎麼選題？**

不是選「最好的」。是選你**認識最多人**的那個。

人最多 = 驗證最快 = 犯錯成本最低。

## 視覺設計

- 第一幕：多個痛點卡片飛入，聚合成一張（動畫）
- 第二幕：物理量畫布 2×2 格子，逐格 build
- 每格用不同顏色（藍/綠/橙/紫）
- 「可以測量」三個字特別強調（放大 + 底線）
- 選題結論用最大字體居中

## 講師備註

> 物理量畫布是整堂課的核心。放慢速度。
> 「這四個格子，是你整個創業過程中最誠實的鏡子。」
> 逐格講解，每格停 5 秒：
> - #1：「說出三個人名。說不出來？先去認識人。」
> - #2：「他們現在花多少錢或時間？如果答案是零——可能不是真痛點。」
> - #3：「今晚你能幫一個人做一次嗎？第一步如果是『寫程式』——重想。」
> - #4：「銀行轉帳也算。Lemon Squeezy 也算。有就好。」
> 「選題不要想太久。認識最多人的那個，就是你的起點。」

## 過場

→ 「你看清了問題。但問題存在，不代表你能做。你需要找到——一個角落。」
