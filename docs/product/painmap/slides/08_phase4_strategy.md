# Slide 08 — Phase 4 策略圖設計（Gagné Event 4：找到位置下）

---

## Action Title

**Phase 4：你的不做清單，比你的功能清單更重要。**

## 內容

Strategy Canvas。一張圖。

橫軸是客戶在意的維度。縱軸是高低。

```
高 │      ╱╲
   │  ╱╲ ╱  ╲        ← 競品：什麼都做。很貴。
   │ ╱  ╲    ╲
低 │╱         ╲
   └──┬──┬──┬──┬──┬──
     價格 易用 功能 速度 支援
```

現在，關鍵來了。

**如果你的曲線跟競品長一樣，只是全部更低——你不是創新者。你只是更爛的版本。**

你要的是：**形狀不同。**

有些比他們高。有些比他們低。有些——根本不做。

---

看一個真實的範例。

| 維度 | 客服 SaaS | 手動 LINE | **你** |
| :--- | :--- | :--- | :--- |
| 價格 | 高 | 低 | **低** |
| 上手速度 | 低 | 高 | **高** |
| 功能數量 | 50+ | 1 | **3** |
| 常見問答 | 有 | 無 | **有** |
| 訂單追蹤 | 有 | 半套 | **不做** |
| 多平台 | 有 | 無 | **不做** |

看到了嗎？

功能數量：3。不是 50。
訂單追蹤：不做。
多平台：不做。

**因為月營收 3 萬的小賣家用 Excel 管訂單就夠了。他們需要的只是不要再重複回答尺寸和運費。**

---

記住這條規則：

> **你的不做清單，必須比你的要做清單長。**
>
> 如果不做比要做少——你砍得不夠。

## 視覺設計

- 第一幕：Strategy Canvas 折線圖（競品曲線）
- 第二幕：你的曲線疊上去——形狀明顯不同
- 第三幕：範例表格，「不做」格子用灰色 + 刪除線
- 最後：規則引用，紅色粗體
- 核心視覺：兩條形狀截然不同的曲線

## 講師備註

> 指著「不做」的格子：「這些灰色的格子，是整張表最有價值的部分。」
> 「每砍掉一個功能，你就省了時間去把核心做到極致。」
> 「不做訂單追蹤——不是因為你做不到。是因為你的客群不需要。」
> 「不做多平台——不是因為技術困難。是因為小賣家 90% 客服在 LINE。」
> 「刻意不做，跟做不到，是完全不同的事。」
